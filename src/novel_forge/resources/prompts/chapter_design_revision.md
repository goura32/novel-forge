# 章設計の改訂

## 目的

レビュー結果に基づき、章設計をスキーマ適合の完成版へ改訂する。章の目的と既存の良い要素を保ちながら、指摘された構成・内容の問題を解消する。

## 応答方針

章設計の改稿者として、レビュー指摘を反映しつつ章全体の因果と感情の流れを整える。未指摘フィールドは不用意に変更しない。

## 実行指示

### 修正順序
1. severity=致命的 を最優先で修正する。
2. severity=重要 を可能な限り修正する。
3. severity=軽微 は品質を損なわない範囲で修正する。

### 注意事項
- レビューで指摘されていない部分は、整合性維持に必要な最小限を除き勝手に変更しない。
- レビューの `before` を `after` に置換する意図を優先しつつ、スキーマ・既存設定・入力情報と矛盾しない完成形に整える。
- レビュー結果や schema の説明用メタ情報を出力 JSON に混入させない。
- `purpose` は `導入` `展開` `転換` `クライマックス` `収束` のいずれか1語のみ。説明文を書かない。
- 出力は章設計そのものにし、issues 等のレビュー用フィールドを含めない。
- 必須フィールドと各シーンの必須要素を欠落させない。
