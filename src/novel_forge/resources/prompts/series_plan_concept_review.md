## レビュー観点
以下の観点を順に評価し、問題があれば `issues` に指摘すること:

- `あらすじ`: 「誰が、何に、どう立ち向かうか」の3要素が本文に含まれているか
- `ジャンル`: ジャンル設定と内容に矛盾がないか
- `世界観`: ルール間に矛盾がないか、明確か
- `ターゲット`: ターゲット読者の記載が具体的か（年齢層・性別・好みのいずれかが明記されているか）
- `言語`: 英語・簡体字・ハングル等の混在がないか
- `言語純度`: 日本語以外の文字（英語・簡体字・ハングル等）が混在していないか

## 出力形式（必須）
以下のJSON配列の形式で必ず出力すること:

```json
{
  "issues": [
    {
      "severity": "致命的",
      "field": "field_name",
      "description": "問題の説明",
      "suggestion": "改善提案",
      "before": "修正前の内容",
      "after": "修正後の内容"
    }
  ]
}
```

**重要**: `issues` は **必ず配列 `[...]`**。オブジェクト `{}` では**いけない**。問題が1件でも3件でも、`["issue_obj", "issue_obj"]` という形式。

**重要**: 評価結果に問題がない場合は、空の `issues` 配列を返すこと。無理に改善点を見つける必要はない。

## 改善案（after）の品質要件

- **プレースホルダー禁止**: `after` には「〇〇」「△△」「XX」「××」「（例）」「（タイトル）」等のプレースホルダーを絶対に含めないこと
- **具体的作品名必須**: ターゲット読者の好みを示す際は、必ず実在する具体的な作品名・タイトルを挙げること（例：『Fate/stay night』『黒執事』『薬屋のひとりごと』等）
- **即採用可能**: `after` の内容はそのまま企画書に採用できる完成形であること
