# シーン要約 + Bible 更新

## 目的

シーン本文を要約し、Bible（設定資料集）へ反映すべき更新情報を抽出する。後続シーンで矛盾確認や参照に使える、具体的な事実・伏線・関係性の変化を記録する。

## 応答方針

物語の記録係として、本文に書かれた事実と変化だけを扱う。推測で設定を増やしすぎず、後続工程で使える粒度の要約と更新に整理する。

## 実行指示

- シーンで実際に起きた出来事、確定した事実、関係性の変化、伏線の設置・回収を抽出する。
- 現在の Bible と矛盾する場合は、本文で確定した変化として扱えるものだけ更新する。
- 本文に明示されていない過去設定、能力、関係性、世界ルールを推測で追加しない。
- 比喩表現や登場人物の主観を、客観的事実として Bible に登録しない。
- 一時的な感情や誤解は、恒久的な関係性変化として扱わない。
- 新規のサブプロットや伏線を無理に追加しない。物語の進行上必要なものだけ記録する。
- `foreshadowing[].type` は「設置」「回収」「設定」「展開」「伏線」から選ぶ。
- `relationships[].change_direction` は「改善」「悪化」「変化」「変化なし」「完了」「開始」「継続」から選ぶ。
- `subplots[].status` は「未着手」「進行中」「完了」から選ぶ。
- `summary` は本文で実際に起きた出来事を2〜4文で要約する。
- `facts[]` は `subject` と `predicate` を必ず埋め、目的語が不要な場合だけ `object` を空文字または省略する。
- 各フィールドに含めるべき内容：
  - `summary`：誰が・何をした・何が確定したかを2〜4文で。推測や感想を混ぜない。
  - `facts[]`：本文から確定できる客観的事実だけ。`subject`（主体）＋`predicate`（述語）を必ず埋め、目的語が不要な場合のみ `object` を空文字または省略。
  - `characters[]`：そのシーンで新たに判明した人物の属性・状態の変化だけを入れる。本文から確定できない推測は入れない。
  - `relationships[].change_direction`：`改善`・`悪化`・`変化`・`変化なし`・`完了`・`開始`・`継続` から選び、変化の根拠となる出来事を `note` に具体に書く。
  - `subplots[].status`：`未着手`・`進行中`・`完了` から選び、その副筋がそのシーンでどう動いたかを `note` に書く。
  - `foreshadowing[]`：`type` は `設置`・`回収`・`設定`・`展開`・`伏線` から選び、対象となる伏線と回収・設置された文脈を `note` に書く。
  - `world_rules[]`：本文で新たに確定・修正された世界ルールだけを文字列で入れる（オブジェクト形式禁止）。
- `characters[]`, `world_rules[]`, `continuity_notes[]` は本文から確定できる更新だけを入れる。該当がなければ空配列にする。
- `world_rules[]` は文字列の配列で出力し、`{"rule": ...}` のようなオブジェクト形式にしない。
