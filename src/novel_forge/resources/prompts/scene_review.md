## 深刻度

- 致命的: 言語制約違反（英語・簡体字・ハングル混入）、POV切り替え、他キャラ内面侵入
- 重要: 論理的破綻、シーン切断（途中で終わる）、2000文字未満
- 軽微: 文体混在、3000文字未満、平板なシーン末尾、表現の微調整、描写の追加提案

## レビュー観点

各 issue の `field` に修正対象のフィールド名を指定し、以下の観点を順に評価。問題があれば `issues` に指摘すること:

- `冒頭フック`: 最初の1-2文に具体的な状況・行動・対立が含まれているか（状況説明だけで始まっていないか）
- `キャラ立ち`: キャラクターが行動・台詞で個性を表しているか
- `感覚描写`: 複数感覚の描写があるか
- `感情描写`: 感情が行動・身体反応で示されているか（Show Don't Tell）
- `シーン末尾`: 末尾に次のシーンへの引き（疑問・緊張・驚き・未解決要素）があるか
- `台詞自然さ`: 台詞が自然で短く、状況を説明していないか
- `文体統一`: 地の文が常体で一貫しているか
- `シーン完結`: シーンが途切れず完結しているか
- `シーン文字数`: 3000文字以上か
- `言語純度`: 英語・簡体字・ハングルが混在していないか
- `POV一貫性`: 視点人物の一貫性が保たれているか、他キャラ内面に侵入していないか
- `論理一貫性`: シーン内の事実・因果関係に矛盾がないか
- `その他`: 上記に当てはまらない指摘

**重要**: 評価結果に問題がない場合は、空の `issues` 配列を返すこと。無理に改善点を見つける必要はない。

## 出版可否ルール

- `ready_for_publication=true` の場合、`致命的` または `重要` の issue は0件でなければならない。
- `ready_for_publication=false` の場合、`致命的` または `重要` の issue を最低1件含める。
- `overall_assessment` には出版可否の理由を1〜2文で書く。
- `strengths` には改稿で消してはいけない良い点を1件以上書く。
- 各 issue の `publication_blocking` は、出版前に必ず直す必要がある場合だけ true にする。
