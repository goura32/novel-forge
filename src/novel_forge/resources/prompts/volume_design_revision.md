## 修正指摘

### 修正順序

1. severity=致命的 を最優先
2. severity=重要 を可能な限り修正
3. severity=軽微 は余力があれば修正

### 修正時の注意

- レビューで指摘されていない部分を勝手に変更しないこと
- 必ず「収束」の章を含めること
- 各シーンの outcome が次のシーンの goal に繋がっていること
- 前巻との整合性を保つこと
- **出力には title、premise、chapters の3つのフィールドが必須**です

## 重要: 出力形式
**以下の3つのフィールドを含む JSON オブジェクトのみを出力してください。他のフィールド（issues 等）は絶対に含めないでください。**

```json
{
  "title": "巻のタイトル（文字列）",
  "premise": "巻のあらすじ（文字列）",
  "chapters": [
    {"title": "章タイトル", "purpose": "導入"},
    {"title": "章タイトル", "purpose": "展開"},
    ...
  ]
}
```

**purpose は必ず `導入` `展開` `転換` `クライマックス` `収束` のいずれか1単語のみ。**
