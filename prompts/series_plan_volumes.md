## 設計要件

- 各巻にタイトル、前提、テーマ、感情の弧、主要イベント、次巻へのフックを設定すること
- 巻全体でシリーズのテーマをカバーすること
- 各巻が独立した物語として完結しつつ、全体として大きなアークを形成すること
- **各巻に具体的な危機・葛藤を設定すること**: 各巻の前提（premise）に、主人公が直面する具体的な危機・葛藤を含めること。曖昧な表現（「闇が絡み合い」「試練に晒される」等）は禁止。具体的な状況・対立・選択を記述すること
- **ジャンル整合性を維持すること**: シリーズのジャンル（心理ミステリー・SF等）から大きく逸脱しないこと。アクション・法廷ドラマ等、異なるジャンルの要素を入れる場合は、シリーズの主要ジャンルとの整合性を説明すること
- **各巻のテーマが重複しないこと**: 各巻のテーマが明確に異なること。似たようなテーマの巻が続かないこと
## 出力

`series_plan_volumes.json` スキーマに適合する JSON を出力すること。

**以下のJSONテンプレートの構造とフィールド名を厳守すること。フィールド名や構造を変更しないこと。**

```json
{
  "planned_volumes": [
    {
      "title": "巻タイトル",
      "premise": "巻のあらすじ",
      "theme": "巻のテーマ",
      "emotional_arc": "巻全体の感情の弧",
      "key_events": ["巻内の主要イベント"],
      "cliffhanger": "次巻へのフック（最終巻以外必須）"
    }
  ]
}
```

**注意**:
- 上記テンプレートのキー名は変更しないこと。値のみを埋めること。
- `planned_volumes` は最低1個以上
- `key_events` は最低1個以上
- `cliffhanger` は最終巻以外必須
