## 設計要件

- 各巻にタイトル、前提、テーマ、感情の弧、主要イベント、次巻へのフックを設定すること
- 巻全体でシリーズのテーマをカバーすること
- 各巻が独立した物語として完結しつつ、全体として大きなアークを形成すること
- **各巻に具体的な危機・葛藤を設定すること**: 各巻の前提（premise）に、主人公が直面する具体的な危機・葛藤を含めること。曖昧な表現（「闇が絡み合い」「試練に晒される」等）は禁止。具体的な状況・対立・選択を記述すること
- **ジャンル整合性を維持すること**: シリーズのジャンル（心理ミステリー・SF等）から大きく逸脱しないこと。アクション・法廷ドラマ等、異なるジャンルの要素を入れる場合は、シリーズの主要ジャンルとの整合性を説明すること
- **各巻のテーマが重複しないこと**: 各巻のテーマが明確に異なること。似たようなテーマの巻が続かないこと
- **巻数の制限**: シリーズは**3巻**とすること。3巻で完結するシリーズ設計をすること。10巻以上の長期シリーズは設計しないこと。

## 巻間の多様性（最重要）

**各巻は固有の目的・危機・解決方法を持つこと。同じパターンを繰り返さないこと。**

- **危機の多様化**: 各巻で主人公が直面する危機の種類を変えること。例:
  - 第1巻: 外部からの脅威（組織・敵対者）
  - 第2巻: 内面的葛藤（信頼の崩壊・選択の代償）
  - 第3巻: 最終決戦（全てを賭けた選択）
- **舞台の多様化**: 各巻で舞台・場所を変えること。同じ場所だけで物語を進めないこと
- **関係性の進展**: 各巻でキャラクター間の関係が段階的に変化すること。同じ関係性のまま巻を跨がないこと
- **解決方法の多様化**: 各巻の危機を解決する方法を変えること。同じ解決策（例: 毎回「料理で解決」）を繰り返さないこと
- **感情の弧の多様化**: 各巻で読者が辿る感情の弧を変えること。例:
  - 第1巻: 緊張→危機→希望
  - 第2巻: 信頼→裏切り→再生
  - 第3巻: 覚悟→決戦→受容

**禁止パターン（以下のような設計は不合格とする）:**
- 全巻で同じ危機（例: 毎回「結界が軋む」）を繰り返す
- 全巻で同じ解決策（例: 毎回「料理で味覚を回復させる」）を使う
- 全巻で同じ感情の弧を辿る
- 巻の終わりが毎回同じパターン（例: 毎回「次巻への伏線」で終わる）
## 出力

`series_plan_volumes.json` スキーマに適合する JSON を出力すること。

**以下のJSONテンプレートの構造とフィールド名を厳守すること。フィールド名や構造を変更しないこと。**

```json
{
  "planned_volumes": [
    {
      "title": "巻タイトル",
      "premise": "巻のあらすじ",
      "theme": "巻のテーマ",
      "emotional_arc": "巻全体の感情の弧",
      "key_events": ["巻内の主要イベント"],
      "cliffhanger": "次巻へのフック（最終巻以外必須）"
    }
  ]
}
```

**注意**:
- 上記テンプレートのキー名は変更しないこと。値のみを埋めること。
- **すべてのフィールドを必ず出力すること。省略禁止。** スキーマに存在するすべてのキーに対応する値を記述すること。
- `planned_volumes` は最低1個以上
- `key_events` は最低1個以上
- `cliffhanger` は最終巻以外必須
