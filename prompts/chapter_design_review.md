## レビュー観点

以下の観点を順に評価し、問題があれば `issues` に指摘すること:

- `タイトル`: 章の内容を適切に表しているか
- `目的`: 導入・展開・転換・クライマックス・収束のいずれかとして適切か
- `テーマ`: 章全体のテーマが明確か
- `感情の弧`: 読者の感情移り変わりが設計されているか
- `結果`: 章の結末が次の章へ繋がるものか
- `シーン構成`: シーンの数・配置・各要素が充実しているか
- `言語純度`: 日本語以外の文字（英語・簡体字・ハングル等）が混在していないか

**重要**: 評価結果に問題がない場合は、空の `issues` 配列を返すこと（`{"issues": []}`）。無理に改善点を見つける必要はない。

## 出力要件（必須）
- `issues` は必ず配列 `[...]` で返すこと。オブジェクト `{}` ではいけない
- 各 issue は必ず以下の6フィールドのみを含めること: `severity`、`field`、`description`、`suggestion`、`before`、`after`
  - `role` 等の追加フィールドは **絶対に含めないこと**
- `before` と `after` は **必ず文字列（string）** で指定すること
  - `null` や `undefined` は絶対に禁止
  - 配列 `[]` やオブジェクト `{}` は絶対に禁止
  - 修正案が複数ある場合は、改行で区切った1つの文字列として出力
- `before` と `after` にプレースホルダー（"〇〇"、"△△"、"XX"、"（例）」等）を含めないこと
- **欠落しているフィールドを指摘する場合、`before` には空文字列 `""` を、`after` には具体的な修正案の文字列を入れること**
- **フィールドが存在する場合、`before` にはそのフィールドの現在値を、`after` には修正後の値を入れること**

## 正しい出力例
```json
{
  "issues": [
    {
      "severity": "重要",
      "field": "theme",
      "description": "テーマフィールドが欠落している",
      "suggestion": "章の中心的なテーマを定義する",
      "before": "",
      "after": "絶望と抵抗の狭間で、歪んだ愛の形を模索する"
    }
  ]
}
```
