# 巻アウトラインの生成（章構成）

## 指示
以下のシリーズ企画をベースに、指定巻の章構成を生成せよ。各シーンの詳細は別途生成するため、ここでは章のタイトルと役割のみを定義する。

## 入力
- シリーズ企画: `{series_plan}`（テキスト形式）
- 巻番号: `{volume_number}`
- ジャンル: `{genre}`
- 出力言語: `{lang}`
- 前巻アウトライン: `{previous_outline}`（第1巻の場合は空文字列）

## 設計原則

1. **物語の弧**: 導入→展開→転換→クライマックス→収束の流れを明確にすること
2. **章の役割**: 各章に明確な役割（導入/展開/転換/クライマックス/収束）を割り当てること
3. **章数**: 5〜6章を目安とすること。**必ず「収束」の章を含めること。** 収束がない場合は不合格とする。4章の場合は「収章」を第4章に兼ねさせること
4. **前巻との整合性**: 前巻アウトラインが提供されている場合、前巻の結果を踏まえた構成にすること

## 出力スキーマ

`volume_design.json` に適合する JSON を出力すること。

```json
{
  "chapters": [
    {
      "title": "章タイトル",
      "purpose": "導入"
    },
    {
      "title": "章タイトル",
      "purpose": "展開"
    },
    {
      "title": "章タイトル",
      "purpose": "転換"
    },
    {
      "title": "章タイトル",
      "purpose": "クライマックス"
    },
    {
      "title": "章タイトル",
      "purpose": "収束"
    }
  ]
}
```

**注意**:
- `purpose` は「導入」「展開」「転換」「クライマックス」「収束」から選択すること。単語のみで記述し、改行や句読点を含めないこと。
- **5〜6章を目安とすること。必ず「収束」の章を含めること。**
- `title` には「第X章」や「Chapter X」などのプレフィックスを付けないこと。章タイトルのみを記述すること。例: 「静寂の代償」
- **前巻アウトラインが提供されている場合、前巻の章タイトルと重複しないこと。**
- **JSONの文字列値内に改行（`\n`）を含めないこと。長い文章は1行で記述すること。**
