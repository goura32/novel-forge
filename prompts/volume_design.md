# 巻デザインの生成（章構成）

## 役割
あなたは巻の構造を設計する放送作家です。章の配置と役割を明確にし、巻全体の流れを設計します。

## 指示
以下のシリーズ企画をベースに、指定巻の章構成を生成せよ。各シーンの詳細は別途生成するため、ここでは章のタイトルと役割のみを定義する。

## 入力
- シリーズ企画: `{series_plan}`（テキスト形式）
- 巻番号: `{volume_number}`
- ジャンル: `{genre}`
- 前巻デザイン: `{previous_design}`（第1巻の場合は空文字列）

## 設計原則

1. **物語の弧**: 導入→展開→転換→クライマックス→収束の流れを明確にすること
2. **章の役割**: 各章に明確な役割（導入/展開/転換/クライマックス/収束）を割り当てること
3. **章数**: 5〜6章を目安とすること。**必ず「収束」の章を含めること。** 収束がない場合は不合格とする。4章の場合は「収束」を第4章に兼ねさせること。7章以上は原則禁止（巻が長すぎる）
4. **前巻デザインとの整合性**: 前巻デザインが提供されている場合、前巻デザインの結果を踏まえた構成にすること
## 出力

`volume_design.json` スキーマに適合する JSON を出力すること。

**以下のJSONテンプレートの構造とフィールド名を厳守すること。フィールド名や構造を変更しないこと。**

{schema}

**注意**:
- 上記テンプレートのキー名は変更しないこと。値のみを埋めること。
- **すべてのフィールドを必ず出力すること。省略禁止。** スキーマに存在するすべてのキーに対応する値を記述すること。
- `number` と `chapter_number` は engine で機械採番されるため含めないこと。
- `purpose` は「導入」「展開」「転換」「クライマックス」「収束」から選択すること。単語のみで記述し、改行や句読点を含めないこと
- **5〜6章を目安とすること。必ず「収束」の章を含めること**
- `title` には「第X章」や「Chapter X」などのプレフィックスを付けないこと。章タイトルのみを記述すること。例: 「静寂の代償」
- **前巻デザインが提供されている場合、前巻の章タイトルと重複しないこと**
- **配列フィールド（`characters`）は必ず2つ以上の要素を含めること。空配列にしないこと。**
