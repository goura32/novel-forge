## 設計要件

- 各キャラクターに性別、年齢、職業、性格、外見、背景、動機、欠点、成長の方向性を明示すること
- キャラクター同士の関係性が自然であること
- 世界観の設定と矛盾しないこと
- **キャラクター間の対立軸を明確にすること**: 主人公と各メインキャラクター間に、具体的な価値観・目標・方法論の対立を設定すること。単なる「意見の違い」ではなく、物語の根幹に関わる対立であること
- **キャラクター間の依存関係を明確にすること**: 主人公が他のキャラクターに何を求め、他のキャラクターが主人公に何を求めているか、具体的に記述すること。相互の依存が物語の動力になること
- **各キャラクターに独自の「行動特性」を設定すること**: 口癖、習慣、癖など、セリフと行動で個性を表現できる具体的な特徴があること。「性格」の抽象的な記述だけでは不十分
## 出力

`series_plan_characters.json` スキーマに適合する JSON を出力すること。

**以下のJSONテンプレートの構造とフィールド名を厳守すること。フィールド名や構造を変更しないこと。**

```json
{
  "main_characters": [
    {
      "name": "キャラクター名（フルネーム）",
      "role": "主人公",
      "arc": "シリーズを通じた成長・変化（200文字以内）",
      "gender": "男性",
      "age": "年齢（例: 28歳）",
      "occupation": "職業・立場",
      "personality": "性格・特徴",
      "appearance": "外見的特徴",
      "background": "過去・経歴",
      "motivation": "動機・目標",
      "flaw": "欠点・弱み",
      "growth": "成長の方向性"
    }
  ]
}
```

**注意**:
- 上記テンプレートのキー名は変更しないこと。値のみを埋めること。
- `role` は「主人公」「ヒロイン」「相棒」「敵対者」「師匠」「仲間」から選択
- `gender` は「男性」「女性」「その他」から選択
- 最低2人以上のキャラクターを設計すること
