# シーンのレビュー

## 指示

以下のシーン本文を評価し、改善点を指摘せよ。本文の新規生成は指示しない。

## 入力

- シーン本文: `{scene}`
- 巻アウトライン: `{outline}`
- コンテキスト: `{context}`
- 進行中のサブプロット: `{subplots}`
- キャラクター関係性: `{relationships}`
- 出力言語: `{lang}`

## 評価カテゴリ

- `opening_hook`: 冒頭のフック。シーンの最初の1-2文で読者の注意を引くか。以下のいずれかで始まっているか:
  - 異常な状況（血を吐く人物が祈りの列にいる）
  - 謎や違和感（見知らぬ感覚、説明できない現象）
  - 緊迫した行動（誰かが倒れている、逃げている）
  - 印象的な感覚描写（異様な臭い、冷たい空気、鐘の音）
  - 会話の途中から始める（「――それは、嘘だ」）
  - **単調な日常描写で始まる場合でも、そのシーンの文脈で自然であれば減点しない。** シーンの位置（導入シーン等）を考慮すること。
  - **減点するのは、シーン全体が平板で読者を引き込む要素が全くない場合のみ。**
- `character_distinction`: キャラ立ち。キャラクターが行動とセリフで個性を示しているか。説明（「彼は冷静な人物だ」）ではなく、行動と会話で示しているか
- `foreshadowing_consistency`: 伏線の整合性。仕込んだ伏線に矛盾はないか。回収されていない伏線が急に消えていないか
- `sensory_coverage`: 五感の網羅。シーンに3つ以上の感覚描写（視覚・聴覚・嗅覚・触覚・味覚）が含まれているか
- `page_turner`: ページターナー。シーンの最後に次のシーンへの興味を引く要素があるか。以下のいずれかで終わっているか:
  - 予想外の事態の発生
  - 新たな謎や問題の提示
  - キャラクターの決意や行動の予兆
  - **シーンの位置（中間シーン等）では、自然な区切りで終わることも許容する。** すべてのシーンに劇的なクロージングを求めるのは不自然。
  - **減点するのは、シーンが唐突に切れたり、読者が次に何が起きるか全く想像できない場合のみ。**
- `dialogue_naturalness`: 台詞の自然さ。キャラクター口調が一貫し、不自然な表現がないか。説明的な台詞（「君は私の妹だ。5年前に離れ離れになった」）がないか
- `tone_consistency`: 文体統一。地の文の文体（ですます調）が一貫しているか。常体（だ・である）が混入していないか。
  - **フィルター単語（「〜と思った」「〜と感じた」「〜が見えた」「〜が聞こえた」）は小説では自然な表現であり、減点対象としない。** ただし、過度に連続して使われている場合は改善点として指摘してよい。
  - **文体の混在（ですます調と常体が混在）のみを減点対象とする。**
- `scene_completeness`: シーン完結。本文が途中で切断されておらず、最後の文が完全な文であるか。「〜よう」「〜ため」「〜が」で止まっていないか
- `language_purity`: 言語純度。英語・簡体字・ハングルが混入していないか。混入がある場合は severity=blocker とすること。
  - **除外**: 日本語で標準的な記号（・、。、「」『』（）！？…ー等）
  - **除外**: 医療・科学分野の英語略語（ICU, DNA, RNA 等）
  - **除外**: 技術用語（CPU, GPU, SSD, USB, URL, HTML 等）
  - **検出対象**: 英語の一般名詞・動詞・形容詞・副詞（the, and, run, beautiful 等）
  - **検出対象**: 簡体字（转、间、门、东、车等）
  - **検出対象**: ハングル（가、나、다等）
- `pov_consistency`: POV 一貫性。視点人物がシーン内で切り替わっていないか。以下のチェックポイントを必ず確認すること:
  - 他キャラクターの内面（思考・感情・感覚）に直接立ち入っていないか
  - 視点人物が見聞きできない情報が描写されていないか
  - 切り替わりがある場合は severity=critical とし、該当箇所を具体的に指摘すること

## 深刻度

- `critical`: 物語の根幹に関わる（POV切り替え、他キャラ内面への侵入、論理的破綻）
- `major`: 品質に大きく影響する（文体混在、英語混入、シーン切断）
- `minor`: 改善点としては望ましいが必須ではない（表現の微調整、描写の追加提案）

## スコアリングガイド（重要 — 必ず従うこと）

`score` は 0-100 の数値で評価する。以下のガイドに従うこと:

- **90-100**: 商業出版レベル。ほぼ問題なし
- **80-89**: 良好。軽微な改善点があるが、そのまま出版可能
- **70-79**: 合格ライン。いくつかの改善点があるが、全体的に読者を引き込む品質
- **60-69**: 改善が必要。複数の major issue がある
- **50-59**: 大幅な改善が必要
- **0-49**: 書き直しが必要

**重要**: 商業出版レベルの小説は 70-85 点の範囲で評価するのが適切。完璧を求めて 60 点以下をつけないこと。フィルター単語や軽微な表現の問題で 60 点以下にするのは不適切。

**スコア計算の考え方**:
- 各 dimension のスコアを 0-100 で評価し、その平均を `score` とする
- `critical` issue がある場合: score 最大 50
- `blocker` issue がある場合: score 最大 30
- `major` issue が 3 つ以上ある場合: score 最大 65
- `minor` issue のみの場合: score 最低 70

## 出力スキーマ

`scene_review.json` に適合する JSON を出力すること。

**重要**: `score` は 0 から 100 の数値で評価すること（10 段階ではない）。
`dimensions[].score` も 0 から 100 の数値で評価すること。

## 出力形式

```json
{
  "score": 0,
  "dimensions": [
    {
      "name": "string",
      "score": 0
    }
  ],
  "issues": [
    {
      "severity": "critical|major|minor|blocker",
      "category": "上記の評価カテゴリから選択 (opening_hook, character_distinction, foreshadowing_consistency, sensory_coverage, page_turner, dialogue_naturalness, tone_consistency, scene_completeness, language_purity, pov_consistency)",
      "description": "string",
      "suggestion": "具体的な修正指示を日本語で記述すること。修正箇所の該当テキストを引用し、どう変えるべきかを明示すること。"
    }
  ],
  "strengths": ["string"],
  "revision_needed": true
}
```

**重要**: `issue.suggestion` は具体的な記述にすること。「修正が必要」という曖昧な記述ではなく、**該当テキストを引用して「この部分を○○に変えよ」と明示すること**。例:
- ❌ 「英語表現を日本語に直してください」
- ✅ 「3行目の「weapon」を「兵器」に置換してください。15行目の「backdoor」を「裏口」に置換してください」
