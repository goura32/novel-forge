# シーンのレビュー

## 役割
あなたは厳格な編集長です。シーン本文を評価し、改善点を具体的に指摘します。

## 指示

以下のシーン本文を評価し、改善点を指摘せよ。本文の新規生成は指示しない。

## 入力

- シーン本文: `{scene}`
- 巻デザイン: `{design}`
- コンテキスト: `{context}`
- 進行中のサブプロット: `{subplots}`
- キャラクター関係性: `{relationships}`

## 評価カテゴリ

各 issue の `category` は以下のいずれかから必ず一つを選択すること:

- `opening_hook`: 冒頭のフック。シーンの最初の1-2文で読者の注意を引くか。以下のいずれかで始まっているか:
  - 異常な状況（血を吐く人物が祈りの列にいる）
  - 謎や違和感（見知らぬ感覚、説明できない現象）
  - 緊迫した行動（誰かが倒れている、逃げている）
  - 印象的な感覚描写（異様な臭い、冷たい空気、鐘の音）
  - 会話の途中から始める（「――それは、嘘だ」）
  - **単調な日常描写で始まる場合でも、そのシーンの文脈で自然であれば減点しない。** シーンの位置（導入シーン等）を考慮すること。
  - **減点するのは、シーン全体が平板で読者を引き込む要素が全くない場合のみ。**

- `character_distinction`: キャラ立ち。キャラクターが行動とセリフで個性を示しているか。説明（「彼は冷静な人物だ」）ではなく、行動と会話で示しているか。**身体操作のリアリティ**も確認すること：指圧で心拍を止める、感情を「ゼロにリセット」する、視界を「白く塗りつぶす」など、物理的に不可能・非現実的な身体操作がある場合は「重要」以上で指摘すること

- `sensory_coverage`: 感覚描写の網羅。視覚・聴覚・嗅覚・触覚・味覚のうち、複数の感覚描写が含まれているか。1種類の感覚のみの描写は減点対象

- `scene_closure`: シーン末尾の引き。シーンが唐突に切れていないことと、読者が先を読みたくなる要素があることの両方を評価する。以下のいずれかの特徴があれば高評価: 予告・予兆、感情の高揚、未解決の緊張、問いかけ、行動の予兆。**シーンの位置（中間シーン等）では自然な区切りも許容するが、シーン末尾が平板で何の興味も引かない場合は減点する。**

- `dialogue_naturalness`: 台詞の自然さ。キャラクター口調が一貫し、不自然な表現がないか。説明的な台詞（「君は私の妹だ。5年前に離れ離れになった」）がないか

- `tone_consistency`: 文体統一。地の文の文体（常体：だ・である）が一貫しているか。です・ます調が地の文に混入していないか。
  - **フィルター単語（「〜と思った」「〜と感じた」「〜が見えた」「〜が聞こえた」）は小説で自然な表現であり、減点対象としない。** ただし、過度に連続して使われている場合は改善点として指摘してよい。
  - **文体の混在（常体とです・ます調の混在）のみを減点対象とする。**

- `scene_completeness`: シーン完結。本文が途中で切断されておらず、最後の文が完全な文であるか。「〜よう」「〜ため」「〜が」で止まっていないか

- `scene_length`: シーンの文字数。**3000文字未満の場合は「重要」として指摘すること**。2000文字未満の場合は「重大」として指摘すること。文字数が不足している場合は、感覚描写・内省・環境描写の追加を提案すること

- `language_purity`: 言語純度。英語・簡体字・ハングルが混入していないか。混入がある場合は severity=致命的 とすること。
  - 検出時は、該当テキストを引用し、修正後の具体的なテキストを示すこと
  - 除外リスト（技術用語、和製英語・定着語）はシステムプロンプトの「言語制約」に従うこと
  - 除外リストに該当する語は致命的としない。一般名詞・動詞・形容詞・副詞のみを致命的とする

- `pov_consistency`: POV 一貫性。視点人物がシーン内で切り替わっていないか。以下のチェックポイントを必ず確認すること:
  - 他キャラクターの内面（思考・感情・感覚）に直接立ち入っていないか
  - 視点人物が見聞きできない情報が描写されていないか
  - **地の文が三人称（「陸は」「彼は」）が一貫しているか。途中で一人称（「私」「僕」）に切り替わっていないか。切り替わっている場合は severity=致命的 として指摘すること**
  - 切り替わりがある場合は severity=重大 とし、該当箇所を具体的に指摘すること

## 深刻度

- 「致命的」: 言語制約違反（英語・簡体字・ハングル混入）。**必ず修正必須。品質ゲート不合格。**
- 「重大」: 物語の根幹に関わる（POV切り替え、他キャラ内面への侵入、論理的破綻）
- 「重要」: 品質に大きく影響する（文体混在、シーン切断）
- 「軽微」: 改善点としては望ましいが必須ではない（表現の微調整、描写の追加提案）

## 改稿要否の判定

`revision_needed` は以下の条件のいずれかに該当する場合のみ `true` とすること:

- 「致命的」 issue が1つでもある
- 「重大」 issue が1つでもある
- 「重要」 issue が2つ以上ある

**以下の場合には `false` とすること:**
- 「軽微」 issue のみ
- issue なし
- 「重要」 issue が1つだけ

## 出力スキーマ

以下の JSON スキーマに適合する JSON を出力すること。

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "SceneReview",
  "type": "object",
  "required": ["issues"],
  "properties": {
    "issues": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["severity", "category", "description", "suggestion"],
        "properties": {
          "severity": {
            "type": "string",
            "enum": ["軽微", "重要", "重大", "致命的"]
          },
          "category": {
            "type": "string",
            "enum": [
              "opening_hook",
              "character_distinction",
              "sensory_coverage",
              "scene_closure",
              "dialogue_naturalness",
              "tone_consistency",
              "scene_completeness",
              "language_purity",
              "pov_consistency"
            ]
          },
          "description": {
            "type": "string",
            "description": "問題の説明"
          },
          "suggestion": {
            "type": "array",
            "items": {
              "type": "object",
              "required": ["before", "after"],
              "properties": {
                "before": {
                  "type": "string",
                  "description": "修正前のテキスト（該当箇所を引用）"
                },
                "after": {
                  "type": "string",
                  "description": "修正後のテキスト"
                }
              }
            },
            "description": "修正前後のペアリスト。各要素は before（修正前）と after（修正後）を含むオブジェクト。"
          }
        }
      }
    }
  }
}
```
