# シーンの改稿

## 指示

以下のシーン本文を、レビュー結果に基づいて改稿せよ。新規生成ではない。

## 入力

- シーン本文: `{scene}`
- レビュー結果: `{review}`（テキスト形式。問題点のリスト、強み、推奨事項を含む）
- 出力言語: `{lang}`

## 改稿方針

1. `blocker` の issue は最優先で必ず修正すること（品質ゲート不合格の直接原因）
2. `critical` の issue は必ず修正すること
3. `major` の issue は可能な限り修正すること
4. `minor` の issue は余力があれば修正すること
5. 原文の良さを活かしつつ、問題点を改善すること
6. レビューで指摘されていない部分を勝手に変更しないこと
7. **言語制約違反の修正（最優先）**: `language_purity` カテゴリの issue がある場合、指摘された英語表現を**すべて**日本語に置換すること。修正漏れがないこと。
   - **簡体字の修正**: 簡体字が検出された場合、日本の常用漢字の字形に置換すること。例: 转→転、间→間、门→門、东→東、车→車、选→選、电→電、话→話
   - **中国語の修正**: 中国語表現が検出された場合、日本語に置換すること。例: 如果有的话→もしあったら、通讯→通信、空氣→空気、选择→選択
   - **英語一般名詞の修正**: 許可リスト（CPU、GPU、SSD、USB、URL、HTML、RAM、ROM、DNS、VPN、SSH、FTP、API、SDK、LAN、WAN、GPS、LED、LCD、OLED、DNA、RNA、CT、MRI、X線、IoT、PC、AI、ICU）以外の英語はすべて日本語に置換すること
8. **文体の維持（最優先）**: 地の文は「です・ます」体を絶対に維持すること。レビューが「常体に変更すべき」と指摘しても、ですます調を維持すること。文体の混在（ですますと常体の混在）を修正することはあっても、文体そのものを常体に変更しないこと。改稿時に誤って常体にしないこと。
9. **POV 一貫性の維持（最優先）**: 改稿時に POV を絶対に切り替えないこと。レビューで POV 違反が指摘された場合、他キャラクターの内面描写を外部観察（行動・表情・台詞）に置き換えること。これが `blocker` または `critical` の issue である場合、最優先で修正すること

## 出力形式

`scene_revision.json` に適合する JSON を出力すること。JSON Schema の各フィールド定義・required・maxLength に従うこと。

- `title`: シーンタイトル
- `content`: 改稿後のシーン本文
- `changes`: 変更内容のリスト。各要素は以下の形式の文字列：
  - `"「[修正前の表現]」を「[修正後の表現]」に置換"` の形式で記述すること
  - 例: `"「weapon を構えた」を「兵器を構えた」に置換"`
  - 例: `"「gaze を向けた」を「視線を注いだ」に置換"`
  - 例: `"「drip 音」を「水滴がぽたりぽたりと音を立てる」に修正"`
  - 複数の変更がある場合は、すべての変更を配列要素として列挙すること
  - **各要素は100字以内に収めること。1つの変更に絞り、詳細は省略すること。**
  - **長い説明や背景の記述は禁止。修正箇所と修正結果のみを簡潔に記述すること。**
