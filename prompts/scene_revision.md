# シーンの改稿

## 指示

以下のシーン本文を、レビュー結果に基づいて改稿せよ。新規生成ではない。

## 入力

- シーン本文: `{scene}`
- レビュー結果: `{review}`（テキスト形式。問題点のリスト、強み、推奨事項を含む）
- 出力言語: `{lang}`

## 改稿方針

### 修正順序（厳守）
1. `severity` が `致命的` の issue を最優先で必ず修正すること（品質ゲート不合格の直接原因）
2. `severity` が `重大` の issue を必ず修正すること
3. `severity` が `重要` の issue を可能な限り修正すること
4. `severity` が `軽微` の issue は余力があれば修正すること

### 修正時の必須処理
- レビュー結果の `issues` 配列に含まれる**すべての issue** を確認すること
- 各 issue の `description` を読み、何が問題かを正確に理解すること
- 各 issue の `suggestion` 配列に記載された修正指示に**すべて**従うこと
- 原文の良さを活かしつつ、問題点を改善すること
- レビューで指摘されていない部分を勝手に変更しないこと
7. **言語制約違反の修正（最優先）**: `language_purity` カテゴリの issue がある場合、指摘された英語表現を**すべて**日本語に置換すること。修正漏れがないこと。
   - **簡体字の修正**: 簡体字が検出された場合、日本の常用漢字の字形に置換すること。例: 转→転、间→間、门→門、东→東、车→車、选→選、电→電、话→話
   - **中国語の修正**: 中国語表現が検出された場合、日本語に置換すること。例: 如果有的话→もしあったら、通讯→通信、空氣→空気、选择→選択
   - **英語一般名詞の修正**: 許可リスト（CPU、GPU、SSD、USB、URL、HTML、RAM、ROM、DNS、VPN、SSH、FTP、API、SDK、LAN、WAN、GPS、LED、LCD、OLED、DNA、RNA、CT、MRI、X線、IoT、PC、AI、ICU）以外の英語はすべて日本語に置換すること
8. **文体の維持（最優先）**: 地の文は「です・ます」体を絶対に維持すること。レビューが「常体に変更すべき」と指摘しても、ですます調を維持すること。文体の混在（ですますと常体の混在）を修正することはあっても、文体そのものを常体に変更しないこと。改稿時に誤って常体にしないこと。
9. **POV 一貫性の維持（最優先）**: 改稿時に POV を絶対に切り替えないこと。レビューで POV 違反が指摘された場合、他キャラクターの内面描写を外部観察（行動・表情・台詞）に置き換えること。これが致命的な issue である場合、最優先で修正すること

## 出力形式

`scene_revision.json` に適合する JSON を出力すること。

**以下のJSONテンプレートの構造とフィールド名を厳守すること。フィールド名や構造を変更しないこと。**

```json
{
  "title": "シーンタイトル（変更しない場合でも入力すること）",
  "content": "改稿後のシーン本文",
  "changes": [{"before": "修正前のテキスト", "after": "修正後のテキスト"}]
}
```

**注意**:
- 上記テンプレートのキー名は変更しないこと。値のみを埋めること。
- `title`: シーンタイトル
- `content`: 改稿後のシーン本文
- `changes`: 修正前後のペアリスト。各要素は before（修正前）と after（修正後）を含むオブジェクト。
  - 例: `{"before": "weapon を構えた", "after": "兵器を構えた"}`
  - 複数の変更がある場合は、すべての変更を配列要素として列挙すること
  - **各要素の before/after はそれぞれ100字以内に収めること。**
  - **長い説明や背景の記述は禁止。修正箇所と修正結果のみを簡潔に記述すること。**
