# シリーズ企画の自己レビュー

## 指示

以下のシリーズ企画テキストを評価し、改善点を指摘せよ。

## 入力

- シリーズ企画:
```
{series_plan}
```
- 出力言語: `{lang}`

## 評価基準

1. **言語純度**: {lang}の文字のみを含み、他言語(中国語・英語等)の混在がないか
   - **除外するもの**: 日本語で標準的に使われる記号（・、。、「」『』（）！？…ー等）は言語制約違反としない
   - **除外するもの**: 日本語の漢字・ひらがな・カタカナは言語制約違反としない
   - **検出対象**: 英語の単語（the, and, is 等）、簡体字（转、间、门等）、ハングル（가、나、다等）
   - **例外**: システムプロンプトの「言語制約」に記載された技術用語・和製英語・定着語は許可される
2. **市場競争力**: 競合との差別化が明確か
3. **ターゲット適合**: 想定読者が具体的か
4. **シリーズ持続性**: 複数巻にわたる展開が見込めるか
5. **タイトル力**: 検索・宣伝に有効か
6. **キャッチコピー**: 読者の興味を引くか

**評価対象について:** 入力テキストに含まれるすべての文字列を評価対象とすること。JSONキー名は含まれていない。

**slug の例外:** `slug` フィールドはローマ字のハイフン区切りパスが許可される。slug のローマ字を言語制約違反として指摘してはならない。

## 出力スキーマ

`series_plan_review.json` に適合する JSON を出力すること。

**以下のJSONテンプレートの構造とフィールド名を厳守すること。フィールド名や構造を変更しないこと。**

```json
{
  "issues": [
    {
      "severity": "重大",
      "category": "カテゴリ名",
      "description": "問題の説明",
      "suggestion": ["改善提案1", "改善提案2"]
    }
  ],
  "strengths": ["強み1", "強み2"],
  "recommendations": ["推奨事項1", "推奨事項2"]
}
```

**注意**:
- 上記テンプレートのキー名（issues, strengths, recommendations）は変更しないこと。
- `issues[].severity` は「重大」「重要」「軽微」から選択すること。
- `issues[].suggestion` は**オブジェクトの配列**であること。各要素は `before`（修正前）と `after`（修正後）を含むオブジェクト。

**suggestion 出力例:**
```json
{
  "severity": "重大",
  "category": "language_purity",
  "description": "シーン本文に英語が混入している",
  "suggestion": [{"before": "weapon を構えた", "after": "兵器を構えた"}, {"before": "gaze を向けた", "after": "視線を注いだ"}]
}
```
