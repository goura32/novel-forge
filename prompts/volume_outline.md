# 巻アウトラインの生成

## 指示

以下のシリーズ企画をベースに、指定巻のアウトライン（章・シーン構成）を生成せよ。

## 入力

- シリーズ企画: `{series_plan}`（テキスト形式。タイトル、あらすじ、ジャンル、ターゲット読者、テーマ、世界観、メインキャラクター、各巻の情報を含む）
- 巻番号: `{volume_number}`
- ジャンル: `{genre}`
- 出力言語: `{lang}`
- 前巻アウトライン: `{previous_outline}`（第1巻の場合は空文字列）

## 設計原則

1. **物語の弧**: 導入→展開→転換→クライマックス→収束の流れを明確にすること
2. **キャラクターアーク**: メインキャラクターに変化（成長・堕落・気づき）があること
3. **ペース配分**: 導入20%、展開・転換50%、クライマックス・収束30%の目安
4. **連続性**: 各シーンの `outcome` が次のシーンの `goal`（State部分）に繋がること
5. **サブプロット**: メイン物語に加えて1〜2つのサブプロットを含めること
6. **伏線**: 次巻への伏線を1箇所以上含めること
7. **章番号の一意性**: 各章の `number` は巻内で一意の連番（1, 2, 3, ...）とする
8. **前巻との整合性**: 前巻アウトラインが提供されている場合、前巻の結果（キャラクターの状態、伏線の回収状況、サブプロットの進捗）を踏まえて整合性のある構成にすること。前巻で回収されていない伏線は本巻で回収するか、次巻に先送りすること

## 出力スキーマ

`volume_outline.json` に適合する JSON を出力すること。

## 各シーンの目標

各シーンには `goal` と `outcome` を分けて設定すること。

- **goal**: シーン開始時の目標。`State: ... | Action: ...` 形式。
  - State: シーン開始時の状況・状態
  - Action: 主人公が取る行動
- **outcome**: シーン終了時の実際の結果。次のシーンの goal（State部分）に繋がること。

**重要**: `goal` と `outcome` は別フィールド。`goal` に予想結果を含めず、`outcome` に実際の結果を書くこと。

例:
- goal: `State: 主人公は王宮の厨房にいる。食材が足りない。 | Action: 主人公は隠し倉庫から特別な香辛料を持ち出す。`
- outcome: `完成した料理で国王の機嫌が直り、主人公は料理長に昇進する。`
