## 執筆ルール

### 1. 文体

地の文は常体（「〜だ。」「〜である。」「〜した。」）に統一すること。

- 「です・ます」調を地の文に使わない
- 台詞・独白はキャラクター口調で自由
- 一文の長さを意識し、短い文と長い文を交互に配置する
- 同じ文末表現を3回以上連続で使わない

### 2. 言語

すべての単語を日本語で記述すること。詳細はシステムプロンプトの「言語制約（最優先）」に従うこと。

- 英語の一般名詞・動詞・形容詞・副詞はすべて日本語に翻訳する
- 英字の人名・地名はカタカナに転記する
- 英単語と日本語の混在パターンは絶対にしない
- 簡体字・繁体字・ハングルは一切使わない
- システムプロンプトに記載された技術用語のみ英語のまま許可

### 3. 描写品質

#### 3a. 感覚描写の網羅
各シーンで**3つ以上の感覚**（視覚・聴覚・嗅覚・触覚・味覚）を描写すること。

- **視覚**: 光の色・強さ・動き、影、表情の変化
- **聴覚**: 環境音、足音、機械音、沈黙
- **嗅覚**: シーンの雰囲気に合った嗅覚描写を含めること。視覚・聴覚に偏りがちなので、意識的に匂いの描写を入れる
- **味覚**: 食事シーンや緊張時の口内感覚。喉の渇き、鉄分のような味、苦味など
- **触覚**: 温度、質感、圧力、痛み

#### 3b. Show Don't Tell（描写せよ、説明するな）
- **悪い例**: 「カイトは怒っていた」「レイナは恐怖を感じていた」
- **良い例**: 「カイトの拳が白くなるほど握りしめられた」「レイナの指先が、かすかに震えていた」
- キャラクターの感情は、行動・表情・身体反応で示すこと

#### 3c. 台詞の自然さ
- 台詞は短く、断片的にする。特に緊迫した状況では短文を多用する
- 台詞で状況を説明しない（「君は私の妹だ。5年前に離れ離れになった」のような説明的台詞を避ける）
- キャラクターごとに口語の癖を付ける。同じ言い回しを全キャラで使い回さない

#### 3d. 陳腐な表現の回避・重複描写の防止
- 「〜の炎が燃え上がる」「〜の波が押し寄せる」「胸の奥が熱くなる」等のクリシェを避ける
- 「運命の〜」「宿命の〜」「闇の中の一筋の光」等の定型的な表現を避ける
- 代わりに、具体的な感覚・行動で表現すること
- **同じ表現・同じ感覚描写を複数シーンで使い回さないこと**。特に以下のパターンは注意:
  - 「朽ちかけた食材が微かに脈打つ」→ 別の表現に変える
  - 「鈴音の眉間が縮む」「鈴音の理性的な防衛が崩れる」→ 別の身体反応で示す
  - 「記憶を代償にする」→ 具体的な記憶の内容を描写し、抽象的な表現を繰り返さない
  - 「結界が軋む」→ 別の表現（「壁面に亀裂が走る」「空間が歪む」等）に変える

#### 3e. 身体操作のリアリティ
キャラクターの身体動作は物理的に可能で現実的なものに限定すること。

- 感情や生理反応を「スイッチ」や「ボタン」のように操作する表現は避ける
- 身体の自然な反応として描写すること
- 例: 「胸の高鳴りを指圧し鎮める」→「胸の高鳴りを手のひらで押さえ、息を整えて鎮める」

### 4. 視点（POV）
シーン設計で指定された POV を厳密に守る。
- 他キャラクターの内面（思考・感情・感覚）に直接立ち入らない
- 視点人物が見聞きできない情報を描写しない
- 地の文は三人称で一貫させること。「私」「僕」などの一人称に切り替えてはならない

### 5. シーン設計の遵守
- シーンの「目標」と「結果」を必ず達成する
- 「葛藤」をシーン内に必ず含める
- 「主要イベント」が指定されている場合は、そのイベントを必ず描写する
- 「舞台設定」が指定されている場合は、その設定を反映する

### 6. 伏線・サブプロット

- 「このシーンで回収すべき伏線」に記載された伏線を必ず回収する。回収方法は明示的であること
- 「進行中のサブプロット」を意識して描写する。サブプロットの進行状況に応じて、該当する要素をシーン内に自然に織り込むこと
- 「キャラクター関係性」に記載された現在の関係性を反映する
- **伏線が「（なし）」の場合でも、シーンの展開に応じて新たな伏線を提示してよい**
- **サブプロットが「（進行中のサブプロットなし）」の場合でも、キャラクター間の関係性変化を描写すること**

### 7. 文字数・構造

- 1シーンあたり **最低3000文字、目標4000-6000文字**を目指す
- **最低3000文字**: これを下回るシーンは品質不足と判断され、レビューで「重要」として指摘される
- 文字数が不足している場合は、感覚描写・内省・環境描写を追加して充実させる
- **ページターナー**: シーンの終わりに読者を引き込む要素（疑問・緊張・驚き）を含める
- **冒頭フック**: 最初の1-2文で読者の注意を引く
- **章の役割の反映**: この章の役割（{chapter_purpose}）を意識した描写を行う

### 8. シーン間の連続性（最優先）

**【最重要】前シーンとの重複を絶対に避けること。**

- `continuity` には前シーンの要約・直近シーン要約・引き継ぎメモ・前シーン全文が含まれる
- **最初に「前シーンの要約」を読み、前シーンの outcome の直後に続く形で書くこと**
- 前シーンで描写した出来事を、同じ表現・類似表現で繰り返してはならない
- 前シーンと同じ状況を別の角度から書き直してはならない（重複禁止）
- 前シーンで終わった場所・時間・状況から、そのまま続きを書くこと
- もし前シーンの outcome と異なる状況から始める場合は、その間の時間経過や移動を明示すること
- **前シーンで既に描写された感覚・感情・行動の描写を、同じキーワードで繰り返さないこと**
  - 例: 前シーンで「朽ちかけた食材が微かに脈打っている」と描写した場合、次シーンでは別の表現（「枯れかけた根菜が微かな光を帯び始める」等）を使う
  - 例: 前シーンで「鈴音の眉間がわずかに縮んだ」と描写した場合、次シーンでは別の身体反応（「鈴音の指先が器の縁を強く握りしめた」等）を使う

## 出力形式

`scene_draft.json` に適合する JSON を出力すること。

**以下のJSONテンプレートの構造とフィールド名を厳守すること。フィールド名や構造を変更しないこと。**

```json
{
  "title": "シーンタイトル",
  "content": "シーンの本文（2000-4000文字）"
}
```

**注意**:
- 上記テンプレートのキー名は変更しないこと。値のみを埋めること。
- **すべてのフィールドを必ず出力すること。省略禁止。** スキーマに存在するすべてのキーに対応する値を記述すること。
- `title` と `content` の両方を必ず出力すること。`title` を空にしないこと。

- `title`: シーンタイトル（必須）。128文字以内。このシーンの内容を要約する簡潔なタイトル。
  - 良い例: 「地下格納庫での対決」「レイナの決意」
  - 悪い例: 「シーン3（第2章）」「戦い」（シーン番号を含む、または抽象的すぎる）
- `content`: シーンの本文（**最低3000文字、目標4000-6000文字**）。章番号やシーンナンバーなどのメタデータは含めない。
  - 本文の冒頭に「シーンX（第Y章）:」や「シーンX:」などのマーカーを付けないこと。本文のみを出力すること。
  - 本文の冒頭にシーンタイトルを付けないこと。タイトルは `title` フィールドに出力すること。
  - `title` と `content` の両方を必ず出力すること。`title` を空にしないこと。
  - **3000文字未満の本文は品質不足と判断される。感覚描写・内省・環境描写を追加して文字数を確保すること。**
  - 地の文は三人称で一貫させること。「私」「僕」などの一人称に切り替えてはならない。
