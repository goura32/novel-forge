# シーン初稿の生成

## 指示

以下のアウトラインとコンテキストから、シーンの初稿を生成せよ。

## 入力

- シリーズ企画: `{series_plan}`
- 巻アウトライン: `{outline}`
- シーン定義: `{scene}`
- コンテキスト: `{context}`
- 前シーン要約（連続性）: `{continuity}`
- 出力言語: `{lang}`

## 品質基準

1. **構造**: 目標→障害→災害→反応→ジレンマ→決断のビートを遵守すること
2. **表現**: Show-don't-tell を徹底し、フィルター単語を排除すること
3. **五感描写**: 視覚・聴覚・嗅覚・触覚・味覚から最低3つ以上の感覚描写を含めること
4. **一貫性**: POV、キャラクター、世界観に矛盾がないこと
5. **重複防止**: シーン内で同一段落や情報が繰り返されないこと

## 読者を惹きつける工夫

- **ホットスタート**: 冒頭から衝撃的なシーン（事件・戦闘・危機・謎）で始めること
- **謎の提示**: 「これは何？」という違和感を提示し、答えを知りたい欲を喚起すること
- **キャラ立ち**: キャラクターの性格は行動とセリフで示すこと（説明しない）
- **伏線の仕込み**: 各シーンで伏線を1つ以上仕込むこと
- **ページターナー**: 章末に次章を読みたくなる仕掛けがあること

## 出力形式

Markdown 形式の本文を出力すること。JSON ではない。

## 文字数制限

1シーンあたり 4,000 字以内とすること。
